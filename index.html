CONTEXTO

Contexto del Proyecto

La administración moderna de las clínicas veterinarias exige la transición de sistemas transaccionales pasivos a ecosistemas predictivos activos. Los registros tradicionales no mitigan de forma proactiva problemáticas como el quiebre de stock de medicamentos o la falta de seguimiento en tratamientos crónicos.

Este proyecto implementa WASIMASI, cuyo valor diferencial radica en el codiseño mediante Agentes de Inteligencia Artificial: un Agente Residente capaz de auditar la base de datos de forma autónoma, procesar historiales médicos y generar reportes predictivos directamente en el dashboard.

DESAFÍO TECNOLÓGICO

Dos Dinámicas de IA

Agentes de Desarrollo
Configurados mediante Google AI Studio con System Prompts estructurados para co-programar el esquema de la base de datos y los endpoints transaccionales del backend.
Agente Residente
Implementado en residentAgent.js (Node.js). Se ejecuta de forma autónoma analizando series temporales de la base de datos e inyectando proyecciones de inventario y métricas estadísticas en el dashboard.

// 1. Gestión de pantallas SPA
function showScreen(screenId) {
  document.querySelectorAll('.screen').forEach(s => s.classList.remove('active'));
  document.getElementById(screenId).classList.add('active');
}

// 2. Submit asíncrono del formulario de atención
document.getElementById('attentionForm').addEventListener('submit', async (e) => {
  e.preventDefault();
  submitBtn.disabled = true;

  const attentionData = {
    idMascota: document.getElementById('formIdMascota').value,
    motivo:    document.getElementById('formMotivo').value,
    diagnostico: document.getElementById('formDiagnostico').value,
    medicacion:  document.getElementById('formMedicacion').value
  };

  try {
    const res = await fetch('/api/atenciones/registrar', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(attentionData)
    });
    const result = await res.json();
    if (result.success) showScreen('screen-clinic-panel');
  } catch (err) {
    alert(`Error: ${err.message}`);
  } finally {
    submitBtn.disabled = false;
  }
});

HITO 3

Lecciones Aprendidas

Qué resolvió la IA de forma brillante y en qué falló — y cómo el equipo lo solucionó.

Lo que la IA resolvió brillantemente
✅
Generación del esquema de BD: A partir del contexto del proyecto, la IA co-programó el script de migración completo con 5 tablas, relaciones, tipos de dato y datos semilla, que el equipo luego optimizó.
✅
Los 10 endpoints del backend: Siguiendo prompts muy específicos (documentados en la Bitácora), la IA generó cada endpoint resolviendo un problema concreto de forma iterativa.
✅
Implementación de WasiBot: La IA definió el patrón del System Prompt inyectado en cada petición, manteniendo al agente enfocado en el dominio veterinario sin necesitar memoria persistente.
Dónde falló y cómo lo resolvieron
🔴
Vulnerabilidad de privacidad (Prompt 3.2.7): El dashboard del dueño exponía las mascotas de todos los usuarios. La IA no lo detectó. El equipo lo corrigió manualmente agregando WHERE id_usuario = :id_usuario con parámetros preparados (PDO).
🟡
Selección de múltiples mascotas (Prompt 3.2.9): Cuando un propietario tenía varias mascotas, el sistema seleccionaba automáticamente la primera. El equipo intervino cambiando la lógica a fetchAll() con modal de selección manual.
🟠
Limitaciones del entorno local (Prompt 3.2.10): La IA no contempló las restricciones de XAMPP. Fue necesario agregar CURLOPT_SSL_VERIFYPEER=false para que WasiBot pudiera conectarse con la API de Gemini. Solución solo para entorno de desarrollo.
Conclusión del Proceso

La IA aceleró significativamente el desarrollo, especialmente en estructura y boilerplate, pero requirió supervisión constante del equipo para corregir errores de lógica de negocio y seguridad que ella no anticipó. El mayor valor no estuvo en confiar ciegamente en el código generado, sino en usar los prompts como una bitácora de decisiones técnicas que permitió iterar rápidamente sobre una base sólida.

PHP
PDO
MySQL

Prompt enviado a la IA:
"Necesito migrar la interfaz estática exportada del ecosistema digital Wasimasi hacia un entorno dinámico local corriendo en XAMPP. Genera el archivo central de conexión conexion.php utilizando PDO para conectarse de forma segura a una base de datos MySQL llamada db_renamascot. Asegúrate de habilitar el manejo de excepciones (PDO::ATTR_ERRMODE) para capturar fallos de red de inmediato."

Resultado: La IA generó el archivo de conexión con PDO, manejo de excepciones y configuración de charset utf8mb4. Primer paso del stack backend.

3.2.7 — 🔐 Vulnerabilidad de Privacidad Corregida
Bug de Privacidad
SQL Filter

Problema detectado por el equipo (no por la IA):
El dashboard del dueño exponía las mascotas de todos los usuarios del sistema.

Prompt de corrección:
"Se ha detectado una vulnerabilidad de privacidad. Modifica el archivo backend api/get_mascotas.php para exigir estrictamente un parámetro $_GET['id_usuario']. Restringe la consulta SQL empleando una cláusula WHERE id_usuario = :id_usuario para asegurar que un cliente autenticado solo tenga visibilidad sobre sus propios animales."

Lección: La IA generó el endpoint sin contemplar el filtro por usuario. Fue el equipo quien identificó la vulnerabilidad y solicitó la corrección.

Dashboard

3.2.9 — ⚠️ Bug de Lógica Corregido
Bug de Lógica
fetchAll()

Problema detectado por el equipo:
Cuando una clínica buscaba registrar atención para un cliente con múltiples mascotas, el sistema seleccionaba automáticamente la primera, impidiendo elegir al paciente correcto.

Prompt de corrección:
"Modifica la lógica del backend api/buscar_paciente_global.php para realizar un fetchAll() que retorne la lista completa de animales asociados al DNI del propietario. Si se detectan múltiples registros, el frontend debe desplegar un listado interactivo en el modal para que el médico seleccione manualmente al paciente."

3.2.10 — WasiBot con Bypass SSL
Gemini 2.5 Flash
cURL
SSL

Prompt enviado a la IA:
"Construye la infraestructura para un Chatbot inteligente embebido. Crea el archivo api/chatbot.php encargado de empaquetar los mensajes del usuario y transmitirlos mediante cURL hacia la API de Google Gemini utilizando el modelo gemini-2.5-flash. Configura las directrices del sistema (systemInstruction) para definir el rol de soporte experto. Añade obligatoriamente CURLOPT_SSL_VERIFYPEER => false para prevenir bloqueos de certificados SSL en XAMPP."

Nota: El CURLOPT_SSL_VERIFYPEER=false es solo para desarrollo local. En producción debe reactivarse la verificación SSL.
